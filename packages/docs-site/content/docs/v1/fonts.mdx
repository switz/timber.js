# Fonts

timber.js handles font loading at the framework level — build-time download, subsetting, and self-hosting for Google Fonts, plus local font support.

## Google Fonts

Import fonts from `@timber/fonts/google`:

```tsx
// app/layout.tsx
import { Inter, JetBrains_Mono } from '@timber/fonts/google';

const inter = Inter({ subsets: ['latin'], weight: ['400', '600'] });
const jetbrains = JetBrains_Mono({ subsets: ['latin'] });

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  );
}
```

At build time, timber.js downloads the font files, subsets them, and serves them from your own domain. No requests to `fonts.googleapis.com` at runtime.

## Local Fonts

Load custom font files with `localFont`:

```tsx
import { localFont } from '@timber/fonts';

const myFont = localFont({
  src: './fonts/MyFont.woff2',
  display: 'swap',
});
```

## Font CSS Variables

Fonts expose CSS variables for use in Tailwind or plain CSS:

```tsx
const inter = Inter({ subsets: ['latin'], variable: '--font-inter' });

// In your layout:
<html className={inter.variable}>

// In CSS:
body { font-family: var(--font-inter); }
```

## Next.js Compatibility

`next/font/google` and `next/font/local` imports are shimmed to timber's font system. Existing code that uses Next.js font imports works without changes.
