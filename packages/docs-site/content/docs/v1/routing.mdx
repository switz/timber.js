# Routing

timber.js uses file-system routing. Files in the `app/` directory become routes. The directory structure is the URL structure.

## Pages and Layouts

A `page.tsx` file makes a route. A `layout.tsx` file wraps all pages in its subtree.

```
app/
  layout.tsx        # Root layout — wraps everything
  page.tsx          # /
  about/
    page.tsx        # /about
  blog/
    layout.tsx      # Wraps all blog pages
    page.tsx        # /blog
    [slug]/
      page.tsx      # /blog/my-post
```

Layouts receive a `children` prop and persist across navigations within their subtree. They don't re-render when you navigate between sibling pages.

## Dynamic Segments

Wrap a directory name in brackets to create a dynamic segment:

```tsx
// app/products/[id]/page.tsx
export default async function ProductPage({ params }) {
  const { id } = await params;
  return <h1>Product {id}</h1>;
}
```

`params` is a Promise — `await` it to access values. This follows the React 19+ convention.

### Catch-All Segments

```
app/docs/[...slug]/page.tsx    # /docs/a, /docs/a/b, /docs/a/b/c
app/docs/[[...slug]]/page.tsx  # Same, plus /docs (zero segments)
```

## Route Groups

Directories wrapped in parentheses group routes without affecting the URL:

```
app/
  (marketing)/
    page.tsx          # /
    about/page.tsx    # /about
  (dashboard)/
    layout.tsx        # Shared dashboard layout
    settings/page.tsx # /settings
```

## Parallel Routes (Slots)

Directories prefixed with `@` are slots — they render simultaneously within a parent layout:

```
app/
  dashboard/
    layout.tsx        # Receives { children, sidebar }
    page.tsx          # Renders as children
    @sidebar/
      page.tsx        # Renders as sidebar prop
      default.tsx     # Fallback when no match
```

```tsx
// app/dashboard/layout.tsx
export default function DashboardLayout({ children, sidebar }) {
  return (
    <div className="flex">
      <aside className="w-64">{sidebar}</aside>
      <main className="flex-1">{children}</main>
    </div>
  );
}
```

Each slot resolves independently — it can have its own error boundaries, layouts, and access gates.

## Intercepting Routes

Intercepting routes show a different component on soft navigation (client-side link click) vs hard navigation (direct URL). This enables the modal pattern:

| Marker | Meaning |
|--------|---------|
| `(.)` | Same level |
| `(..)` | One level up |
| `(...)` | Root level |

## Private Folders

Prefix a directory with `_` to exclude it from routing:

```
app/
  _components/    # Not a route — co-locate shared code here
    Button.tsx
  page.tsx        # /
```

## API Routes

Export HTTP method handlers from `route.ts`:

```ts
// app/api/hello/route.ts
export async function GET(ctx) {
  return Response.json({ message: 'Hello' });
}
```
