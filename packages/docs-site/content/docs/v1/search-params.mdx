# Search Params

Every route can declare typed search params via a co-located `search-params.ts` file. The framework parses them on the server, syncs them on the client, and generates TypeScript types at build time.

## Defining Search Params

```ts
// app/products/search-params.ts
import { createSearchParams } from '@timber/app/search-params';
import { z } from 'zod/v4';

export const searchParams = createSearchParams({
  page: z.coerce.number().default(1),
  sort: z.enum(['price', 'name', 'date']).default('date'),
  q: z.string().optional(),
});
```

## Server Usage

In `page.tsx`, `middleware.ts`, and `access.ts`, search params are auto-parsed via ALS:

```tsx
// app/products/page.tsx
import { searchParams } from '@timber/app/server';

export default async function ProductsPage() {
  const { page, sort, q } = await searchParams();
  const products = await getProducts({ page, sort, query: q });

  return <ProductList products={products} />;
}
```

Values are validated and typed. Invalid params fall back to defaults.

## Client Usage

Use `useQueryStates` to read and update search params from client components:

```tsx
'use client';
import { useQueryStates } from '@timber/app/search-params';
import { searchParams } from './search-params';

export function ProductFilters() {
  const [params, setParams] = useQueryStates(searchParams);

  return (
    <div>
      <input
        value={params.q ?? ''}
        onChange={(e) => setParams({ q: e.target.value })}
      />
      <select
        value={params.sort}
        onChange={(e) => setParams({ sort: e.target.value })}
      >
        <option value="date">Date</option>
        <option value="price">Price</option>
        <option value="name">Name</option>
      </select>
    </div>
  );
}
```

By default, `useQueryStates` triggers an RSC navigation when params change — the server re-renders with the new values. Set `shallow: true` to update the URL without a server roundtrip.

## Codecs

Search params accept any codec that implements `SearchParamCodec<T>`:

- **Zod schemas** — `z.coerce.number()`, `z.enum([...])`, etc.
- **nuqs parsers** — `parseAsInteger`, `parseAsString`, etc.
- **Custom codecs** — any object with `parse(string): T` and `serialize(T): string`

## URL Key Aliasing

Map a property name to a different URL key:

```ts
export const searchParams = createSearchParams({
  page: { codec: z.coerce.number().default(1), key: 'p' },
  query: { codec: z.string().optional(), key: 'q' },
});
```

`/products?p=2&q=shoes` maps to `{ page: 2, query: 'shoes' }`.

## Composition

Extend or pick from existing definitions:

```ts
const base = createSearchParams({ page: z.coerce.number().default(1) });
const extended = base.extend({ sort: z.enum(['asc', 'desc']).default('asc') });
const subset = extended.pick('page');
```
