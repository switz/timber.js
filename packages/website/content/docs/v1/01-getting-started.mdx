# Getting Started

timber.js is a React web framework built on Vite. It runs natively on Cloudflare Workers, Node.js, Bun, and many other platforms.

It provides an emphasis on clarity, developer experience, and web standards. It's designed to be deployed on server and serverless platforms, but you can build for static even deploy websites with: *zero-client-javascript*.

It takes inspiration from next.js and a long history of web frameworks, but redirects intentions towards a different set of values.

With Timber you get:

* Proper Status Code Handling
* Graceful Degradation w/ Javascript Disabled
* Real Middleware
* Fully typed and integrated schemas that work across the server and the client, for:
  * search params
  * url params
  * cookies

Timber embraces the inherent complexity of building a website that sits comfortably across the server-and-client boundary, but it offers very little _magic_. You will understand how and why your websites work with less confusion.

It is an attempt to give developers the autonomy to write websites to perform how they want them to. It doesn't make those decisions for you and works much more how you would expect a web framework to.

## Create a Project

```bash
pnpm create @timber/app fun-app
cd fun-app
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000). You have a running app.

## Project Structure

A new timber.js project looks like this:

```
fun-app/
  src/
    app/
      layout.tsx        # Root layout — wraps every page
      page.tsx          # Home page (/)
      globals.css
  timber.config.ts    # Framework configuration
  vite.config.ts      # Vite configuration
  package.json
  tsconfig.json
```

The `app/` directory is your route tree. Files named `page.tsx` become pages. Files named `layout.tsx` wrap their children. This should feel familiar if you've used Next.js — the key differences are in how rendering, status codes, and streaming work under the hood.

## Your First Page

```tsx
// app/page.tsx
export default async function HomePage() {
  return (
    <div>
      <h1>Hello, timber.js</h1>
      <p>This page was server-rendered with a real 200 status code.</p>
    </div>
  );
}
```

Pages are async server components by default.

## Adding a Second Route

Create `app/about/page.tsx`:

```tsx
// app/about/page.tsx
export default async function AboutPage() {
  return <h1>About</h1>;
}
```

Navigate to `/about`. The file system _is_ the router.
