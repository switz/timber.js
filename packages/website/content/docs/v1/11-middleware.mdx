# Middleware

timber.js has two middleware layers: `proxy.ts` for global concerns and `middleware.ts` for per-segment logic.

## `proxy.ts` — Global Middleware

`app/proxy.ts` runs on every request before route matching. Use it for CORS, rate limiting, security headers, and request logging.

```ts
// app/proxy.ts — array form
export default [
  cors({ origins: ['https://example.com'] }),
  rateLimit({ requests: 100, window: '1m' }),
  securityHeaders(),
];
```

```ts
// app/proxy.ts — function form
export default function proxy(req: Request, next: () => Promise<Response>) {
  const start = performance.now();
  const res = await next();
  console.log(`${req.method} ${req.url} — ${performance.now() - start}ms`);
  return res;
}
```

`proxy.ts` has `next()` — it can wrap the entire request lifecycle. This is where you observe and modify responses.

## `middleware.ts` — Per-Segment Middleware

`middleware.ts` files run after route matching, before rendering. They're scoped to their route segment.

```ts
// app/dashboard/middleware.ts
import { redirect } from '@timber-js/app/server';

export default async function middleware(ctx) {
  const session = await getSession(ctx);
  if (!session) redirect('/login');
}
```

**Key difference from proxy.ts:** Middleware does not have `next()`. It's a gate — it either short-circuits (redirect, deny, return a Response) or passes through silently. It cannot observe or modify the render result.

**Key difference from Next.js:** Next.js has a single global `middleware.ts`. timber.js lets you place `middleware.ts` at any route segment. Each runs in order from root to leaf.

## Middleware Context

```ts
export default async function middleware(ctx: MiddlewareContext) {
  ctx.params;        // Route params
  ctx.request;       // The incoming Request
  ctx.url;           // Parsed URL
  ctx.searchParams;  // Typed search params (if search-params.ts exists)
}
```

## Execution Order

For a request to `/dashboard/settings`:

1. `proxy.ts` (global)
2. `app/middleware.ts` (root, if exists)
3. `app/dashboard/middleware.ts` (segment)
4. `app/dashboard/settings/middleware.ts` (segment)
5. `access.ts` checks (see [Authorization](/docs/v1/authorization))
6. Render

Any middleware can short-circuit the chain.
