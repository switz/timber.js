# Data Fetching

In timber.js, your components are your data layer. Server components are async — fetch data directly in the component that needs it.

## The Basics

```tsx
// app/products/[id]/page.tsx
import { deny } from '@timber/app/server';

export default async function ProductPage({ params }) {
  const { id } = await params;
  const product = await db.products.findUnique({ where: { id } });

  if (!product) deny(404);

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.description}</p>
    </div>
  );
}
```

No loaders. No `getServerSideProps`. No `useEffect`. The component runs on the server, fetches what it needs, and renders HTML.

## No Implicit Caching

`fetch()` is never patched or cached. A bare `fetch()` call hits the network every time. If you want caching, use `timber.cache()` — see [Caching](/docs/v1/caching).

```tsx
// This hits the API every request
const data = await fetch('https://api.example.com/data').then(r => r.json());

// This is cached for 60 seconds
const getData = cache(
  async () => fetch('https://api.example.com/data').then(r => r.json()),
  { ttl: 60 }
);
```

## Parallel Data Fetching

Fetch multiple things concurrently with `Promise.all`:

```tsx
export default async function DashboardPage() {
  const [user, stats, notifications] = await Promise.all([
    getUser(),
    getStats(),
    getNotifications(),
  ]);

  return (
    <div>
      <UserHeader user={user} />
      <StatsGrid stats={stats} />
      <NotificationList items={notifications} />
    </div>
  );
}
```

## Streaming Secondary Content

Primary content fetches outside `<Suspense>` — it can affect the status code. Secondary content goes inside `<Suspense>` — it streams after the shell flushes.

```tsx
import { Suspense } from 'react';

export default async function ProductPage({ params }) {
  const { id } = await params;
  const product = await getProduct(id);
  if (!product) deny(404);

  return (
    <div>
      <ProductHeader product={product} />
      <Suspense fallback={<ReviewsSkeleton />}>
        <ProductReviews productId={product.id} />
      </Suspense>
    </div>
  );
}
```

`getProduct` runs before the flush — if it fails, you get a real 404. Reviews stream in after.

## `React.cache` for Request Deduplication

If multiple components in the same request need the same data, use `React.cache`:

```tsx
import { cache } from 'react';

export const getUser = cache(async (id: string) => {
  return db.users.findUnique({ where: { id } });
});
```

This deduplicates within a single request. For cross-request caching, use `timber.cache()`.
