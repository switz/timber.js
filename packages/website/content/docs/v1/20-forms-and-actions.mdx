# Forms & Server Actions

Forms in timber.js work without JavaScript. They submit to the server, validate, and return a result. When JavaScript is available, they enhance with client-side state and optimistic updates.

## Basic Server Action

```ts
// app/todos/actions.ts
'use server';
import { z } from 'zod/v4';
import { action } from '@/lib/action';
import { revalidatePath } from '@timber/app/cache';

export const createTodo = action
  .schema(z.object({ title: z.string().min(1) }))
  .action(async ({ input, ctx }) => {
    await db.todos.create({ ...input, userId: ctx.user.id });
    return revalidatePath('/todos');
  });
```

## Action Clients

Define reusable action middleware with `createActionClient`. This handles auth, validation, and error handling in one place:

```ts
// lib/action.ts
import { createActionClient, ActionError } from '@timber/app/server';
import { getUser } from '@/lib/auth';

export const action = createActionClient({
  middleware: async () => {
    const user = await getUser();
    if (!user) throw new ActionError('UNAUTHORIZED');
    return { user };
  },
});

export const adminAction = createActionClient({
  middleware: async () => {
    const user = await getUser();
    if (!user) throw new ActionError('UNAUTHORIZED');
    if (!user.isAdmin) throw new ActionError('FORBIDDEN');
    return { user };
  },
});
```

Route middleware (`middleware.ts`) does not run for server actions. Auth for actions goes through `createActionClient`.

## Using Actions in Forms

```tsx
'use client';
import { useActionState } from 'react';
import { createTodo } from './actions';

export default function TodoForm() {
  const [state, action, pending] = useActionState(createTodo, null);

  return (
    <form action={action}>
      <input name="title" required />
      <button type="submit" disabled={pending}>
        {pending ? 'Adding...' : 'Add Todo'}
      </button>
      {state?.error && <p className="text-red-600">{state.error}</p>}
    </form>
  );
}
```

This form works with or without JavaScript. Without JS, it submits as a standard form POST. With JS, `useActionState` handles pending state and error display.

## After Mutation

Three patterns for updating the UI after a mutation:

| Pattern | When to use |
|---------|------------|
| `redirect('/path')` | User should land somewhere new |
| `revalidatePath('/path')` | Current page needs fresh data |
| `useOptimistic` | Instant UI update, action confirms in background |

## Validation

The `.schema()` method accepts any Standard Schema-compatible validator — Zod, Valibot, or ArkType:

```ts
export const updateProfile = action
  .schema(z.object({
    name: z.string().min(1).max(100),
    bio: z.string().max(500).optional(),
  }))
  .action(async ({ input, ctx }) => {
    await db.users.update({ where: { id: ctx.user.id }, data: input });
    return revalidatePath('/profile');
  });
```

Validation errors are returned to the client automatically. Failed form submissions are returned with the original input so users don't lose their work.
