# Client Navigation

timber.js supports client-side navigation via the `<Link>` component. Clicking a link fetches an RSC payload from the server and reconciles the DOM without a full page reload.

## `<Link>`

```tsx
import { Link } from '@timber/app/client';

<Link href="/about">About</Link>
<Link href="/products/123">Product</Link>
<Link href="/dashboard" replace>Dashboard</Link>
```

`<Link>` type-checks `href` against the generated route map. Invalid routes produce a TypeScript error.

## `useRouter`

For programmatic navigation:

```tsx
'use client';
import { useRouter } from '@timber/app/client';

export function LogoutButton() {
  const router = useRouter();

  async function handleLogout() {
    await logout();
    router.push('/login');
  }

  return <button onClick={handleLogout}>Log out</button>;
}
```

### Methods

| Method | Description |
|--------|-------------|
| `router.push(href)` | Navigate to a new URL |
| `router.replace(href)` | Navigate without adding a history entry |
| `router.refresh()` | Re-fetch the current route's RSC payload |
| `router.back()` | Go back in history |
| `router.forward()` | Go forward in history |

## `usePathname`

Returns the current pathname:

```tsx
'use client';
import { usePathname } from '@timber/app/client';

export function NavLink({ href, children }) {
  const pathname = usePathname();
  const isActive = pathname === href;

  return (
    <Link href={href} className={isActive ? 'font-bold' : ''}>
      {children}
    </Link>
  );
}
```

## `useSelectedLayoutSegment`

Returns the active segment within a layout — useful for highlighting nav items:

```tsx
'use client';
import { useSelectedLayoutSegment } from '@timber/app/client';

export function DashboardNav() {
  const segment = useSelectedLayoutSegment();
  // segment is "settings", "projects", etc.
}
```

## `useLinkStatus`

Track per-link pending state during navigation:

```tsx
'use client';
import { Link, useLinkStatus } from '@timber/app/client';

export function NavItem({ href, children }) {
  return (
    <Link href={href}>
      <NavItemInner>{children}</NavItemInner>
    </Link>
  );
}

function NavItemInner({ children }) {
  const { pending } = useLinkStatus();
  return <span className={pending ? 'opacity-50' : ''}>{children}</span>;
}
```

## Scroll Restoration

timber.js takes manual control of scroll restoration during client-side navigation. The browser's native scroll restoration doesn't work with React's document-level rendering, so timber handles it explicitly.

### Default behavior

- **Forward navigation** scrolls to the top of the page.
- **Back/forward** restores the saved scroll position.
- Scroll positions are saved in `history.state` so each history entry remembers its own position, even if two entries share the same URL.

### Disabling scroll-to-top

Pass `scroll={false}` to `<Link>` to preserve the current scroll position after navigation — useful for tab-style UIs where the content changes but the page shouldn't jump:

```tsx
<Link href="/dashboard/settings" scroll={false}>
  Settings
</Link>
```

Or programmatically:

```tsx
router.push('/dashboard/settings', { scroll: false });
```

### Custom scroll containers

By default, timber restores scroll on the window and the document root elements (`<html>` and `<body>`). If your layout uses a custom scrollable container (e.g. a `<main>` with `overflow-y: auto`), add the `data-timber-scroll-restoration` attribute so timber can manage its scroll position:

```tsx
<main className="overflow-y-auto h-screen" data-timber-scroll-restoration>
  {children}
</main>
```

This ensures scroll-to-top on forward navigation and scroll restoration on back/forward work correctly within that container.

> **Tip:** Prefer window-level scrolling when possible. Use sticky positioning for fixed sidebars and navs instead of constraining the page height and scrolling a nested container.

## `Link.onNavigate`

Intercept navigation for custom logic like view transitions:

```tsx
<Link
  href="/gallery"
  onNavigate={(e) => {
    if (document.startViewTransition) {
      e.preventDefault();
      document.startViewTransition(() => e.navigate());
    }
  }}
>
  Gallery
</Link>
```
