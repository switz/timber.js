# Content Collections

Content collections let you define typed, file-based content in a `content/` directory. Each collection has a Zod schema for frontmatter validation and a transform for processing (like MDX compilation).

## Defining a Collection

```ts
// content-collections.ts
import { defineCollection, defineConfig } from '@content-collections/core';
import { compileMDX } from '@content-collections/mdx';
import { z } from 'zod/v4';

const docs = defineCollection({
  name: 'docs',
  directory: 'content/docs',
  include: '**/*.{mdx,md}',
  schema: z.object({
    title: z.string(),
    description: z.string(),
    order: z.number(),
    section: z.string().optional(),
  }),
  transform: async (document, context) => {
    const mdx = await compileMDX(context, document);
    return { ...document, mdx };
  },
});

export default defineConfig({ collections: [docs] });
```

## Using Collections

Import the generated collection and query it:

```tsx
import { allDocs } from 'content-collections';

export default async function DocsPage({ params }) {
  const { slug } = await params;
  const doc = allDocs.find((d) => d._meta.fileName === slug);
  if (!doc) deny(404);

  return (
    <article className="prose">
      <h1>{doc.title}</h1>
      <doc.mdx.default />
    </article>
  );
}
```

## Content Directory Structure

```
content/
  docs/
    v1/
      getting-started.mdx
      routing.mdx
  blog/
    hello-world.mdx
```

Each file's `_meta` includes `path`, `fileName`, and `directory` — useful for filtering by version, building sidebar navigation, or generating static params.

## MDX Components

Customize MDX rendering with `mdx-components.tsx`:

```tsx
// mdx-components.tsx
import type { MDXComponents } from 'mdx/types';

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components,
    h1: ({ children }) => <h1 className="text-3xl font-bold">{children}</h1>,
    code: ({ children }) => <code className="bg-gray-100 rounded px-1">{children}</code>,
  };
}
```

## Type Generation

Content collections generate TypeScript types at build time. Every document is fully typed — frontmatter fields, computed properties from transforms, and `_meta` information.
