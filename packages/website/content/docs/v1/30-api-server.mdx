# @timber/app/server

Server-side utilities and types. Import from `@timber/app/server`.

## Navigation & Status

### `deny(status?)`

Produce an HTTP error status and render the matching status file. Default status is 403.

```ts
import { deny } from '@timber/app/server';

deny();      // 403 Forbidden
deny(404);   // 404 Not Found
deny(401);   // 401 Unauthorized
```

### `redirect(url, type?)`

Redirect to a URL. Sends HTTP 307 (temporary) by default.

```ts
import { redirect } from '@timber/app/server';

redirect('/login');              // 307 Temporary Redirect
redirect('/new-page', 'permanent'); // 308 Permanent Redirect
```

### `permanentRedirect(url)`

Shorthand for `redirect(url, 'permanent')`. Sends HTTP 308.

### `notFound()`

Alias for `deny(404)`.

### `redirectExternal(url)`

Redirect to an external URL. Same behavior as `redirect()` but allows absolute URLs outside the app.

## Request Context

### `headers()`

Returns the incoming request headers. ALS-backed — works in server components, middleware, access checks.

```ts
import { headers } from '@timber/app/server';

const h = headers();
const auth = h.get('authorization');
```

Returns `ReadonlyHeaders` — a read-only `Headers` interface.

### `cookies()`

Returns the request cookies with mutation methods.

```ts
import { cookies } from '@timber/app/server';

const c = cookies();
c.get('session');           // Read
c.set('theme', 'dark');     // Set (before flush only)
c.delete('session');        // Delete
c.clear();                  // Clear all
```

Mutations are only allowed before the response flushes. Attempting to set a cookie after flush throws in dev mode.

### `searchParams()`

Returns typed, validated search params when a `search-params.ts` file exists for the current route.

```ts
import { searchParams } from '@timber/app/server';

const { page, sort } = await searchParams();
```

## Actions

### `createActionClient(options)`

Create a reusable action client with middleware for auth and validation.

```ts
import { createActionClient, ActionError } from '@timber/app/server';

const action = createActionClient({
  middleware: async () => {
    const user = await getUser();
    if (!user) throw new ActionError('UNAUTHORIZED');
    return { user };
  },
});

// Usage:
const myAction = action
  .schema(z.object({ title: z.string() }))
  .action(async ({ input, ctx }) => {
    // ctx.user is typed from middleware return
  });
```

### `ActionError`

Throw from action middleware or bodies to produce a typed error result.

```ts
throw new ActionError('UNAUTHORIZED');
throw new ActionError('VALIDATION_FAILED', { field: 'email' });
```

### `revalidatePath(path)`

Re-render a path and return the RSC payload as part of the action response.

### `revalidateTag(tag)`

Invalidate all cache entries with the given tag.

## Errors

### `RenderError(code, data?, status?)`

Throw structured error data that survives the RSC → client boundary.

```ts
import { RenderError } from '@timber/app/server';

throw new RenderError('PRODUCT_NOT_FOUND', {
  title: 'Product not found',
  resourceId: params.id,
});
```

## Types

### `RouteContext`

```ts
interface RouteContext {
  params: Record<string, string>;
  request: Request;
  url: URL;
}
```

### `MiddlewareContext`

```ts
interface MiddlewareContext {
  params: Record<string, string>;
  request: Request;
  url: URL;
  searchParams: T; // Typed if search-params.ts exists
}
```

### `AccessContext`

```ts
interface AccessContext {
  params: Record<string, string>;
  searchParams: T;
}
```

### `Metadata`

The full metadata type — see [Metadata & SEO](/docs/v1/metadata) for usage.

### `MetadataRoute`

Type for metadata route files (sitemap, robots, etc.).

## Lifecycle

### `waitUntil(promise)`

Register a promise to be kept alive after the response is sent. Platform-dependent — works on Cloudflare Workers and other platforms that support it.

```ts
import { waitUntil } from '@timber/app/server';

waitUntil(analytics.track(event));
```
