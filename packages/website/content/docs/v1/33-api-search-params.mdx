# @timber-js/app/search-params

Typed search parameter definitions and URL synchronization. Import from `@timber-js/app/search-params`.

## `createSearchParams(definition)`

Create a typed search params definition for a route.

```ts
import { createSearchParams } from '@timber-js/app/search-params';
import { z } from 'zod/v4';

export const searchParams = createSearchParams({
  page: z.coerce.number().default(1),
  sort: z.enum(['price', 'name', 'date']).default('date'),
  q: z.string().optional(),
});
```

### Returns: `SearchParamsDefinition<T>`

| Method | Description |
|--------|-------------|
| `.extend(defs)` | Add additional params |
| `.pick(...keys)` | Create a subset |
| `.serialize(values)` | Convert typed values to URL search string |
| `.href(base, values)` | Build a full URL with params |
| `.toSearchParams(values)` | Convert to `URLSearchParams` |
| `.parse(searchParams)` | Parse and validate from `URLSearchParams` or string |

## `SearchParamCodec<T>`

The codec protocol. Any object with `parse` and `serialize` satisfies it:

```ts
interface SearchParamCodec<T> {
  parse(value: string | undefined): T;
  serialize(value: T): string | undefined;
}
```

Compatible codecs:
- **Zod schemas** — `z.coerce.number()`, `z.enum([...])`, etc.
- **nuqs parsers** — `parseAsInteger`, `parseAsString`, `parseAsBoolean`
- **Custom** — any object implementing the interface

## `fromSchema(schema)`

Bridge a Zod or Standard Schema to `SearchParamCodec`:

```ts
import { fromSchema } from '@timber-js/app/search-params';

const pageCodec = fromSchema(z.coerce.number().default(1));
```

## `fromArraySchema(schema)`

Bridge for array-valued search params:

```ts
const tagsCodec = fromArraySchema(z.array(z.string()));
// Parses ?tags=a&tags=b → ['a', 'b']
```

## URL Key Aliasing

Map property names to different URL keys:

```ts
createSearchParams({
  page: { codec: z.coerce.number().default(1), key: 'p' },
  query: { codec: z.string().optional(), key: 'q' },
});
// ?p=2&q=shoes → { page: 2, query: 'shoes' }
```

## `useQueryStates(definition, options?)`

Client hook for reading and updating typed search params. Backed by nuqs.

```tsx
'use client';
import { useQueryStates } from '@timber-js/app/search-params';

const [params, setParams] = useQueryStates(searchParams);
setParams({ page: 2 });
setParams({ sort: 'price' }, { shallow: true });
```

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `shallow` | `boolean` | `false` | `true`: URL-only update. `false`: triggers RSC navigation |
| `scroll` | `boolean` | `true` | Scroll to top on change |
| `history` | `'push' \| 'replace'` | `'push'` | History mode |

## Build-Time Analysis

### `analyzeSearchParams(filePath)`

Static analysis of `search-params.ts` files at build time for route map type generation. Used internally by the build system.
