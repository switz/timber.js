# timber.config.ts

Project-wide configuration. Export a default object from `timber.config.ts` at the project root.

## Full Type

```ts
interface TimberUserConfig {
  output?: 'server' | 'static';
  clientJavascript?: boolean | ClientJavascriptConfig;
  adapter?: TimberPlatformAdapter;
  cacheHandler?: CacheHandler;
  allowedOrigins?: string[];
  csrf?: boolean;
  limits?: {
    actionBodySize?: string;
    uploadBodySize?: string;
    maxFields?: number;
  };
  pageExtensions?: string[];
  dev?: {
    slowPhaseMs?: number;
  };
  cookies?: {
    secret?: string;
    secrets?: string[];
  };
  mdx?: {
    remarkPlugins?: any[];
    rehypePlugins?: any[];
    recmaPlugins?: any[];
    remarkRehypeOptions?: any;
  };
}
```

## Options

### `output`

- **Type:** `'server' | 'static'`
- **Default:** `'server'`

Server mode runs on every request. Static mode pre-renders at build time.

### `clientJavascript`

- **Type:** `boolean | { disabled: boolean; enableHMRInDev?: boolean }`
- **Default:** `true`

Control client-side JavaScript output. Set to `false` to disable all client JS. Use the object form to disable client JS in production while keeping HMR in development:

```ts
clientJavascript: { disabled: true, enableHMRInDev: true }
```

### `adapter`

- **Type:** `TimberPlatformAdapter`
- **Default:** none

Deployment adapter. See [Deployment](/docs/v1/deployment).

```ts
import { cloudflare } from '@timber-js/app/adapters/cloudflare';
import { nitro } from '@timber-js/app/adapters/nitro';

adapter: cloudflare();
adapter: nitro({ preset: 'node-server' });
```

### `cacheHandler`

- **Type:** `CacheHandler`
- **Default:** `MemoryCacheHandler`

Cache backend for `timber.cache()`. See [Cache API](/docs/v1/api-cache).

### `allowedOrigins`

- **Type:** `string[]`
- **Default:** none

Origins allowed for CORS and CSRF validation.

### `csrf`

- **Type:** `boolean`
- **Default:** `true` (in server mode)

Enable CSRF protection for server actions and form submissions.

### `limits`

Control request body size limits:

```ts
limits: {
  actionBodySize: '1mb',   // Server action body limit
  uploadBodySize: '10mb',  // File upload body limit
  maxFields: 1000,         // Maximum FormData fields
}
```

### `pageExtensions`

- **Type:** `string[]`
- **Default:** `['tsx', 'ts', 'jsx', 'js']`

File extensions recognized as pages and routes. Add `'mdx'` for MDX pages.

### `dev.slowPhaseMs`

- **Type:** `number`
- **Default:** framework default

Threshold in milliseconds for slow-phase warnings in dev mode.

### `cookies`

Cookie signing configuration:

```ts
cookies: {
  secret: 'my-secret-key',
  // Or rotate secrets:
  secrets: ['new-secret', 'old-secret'],
}
```

## `TimberPlatformAdapter`

The adapter interface for community adapters:

```ts
interface TimberPlatformAdapter {
  name: string;
  buildOutput(config: TimberConfig, buildDir: string): Promise<void>;
  preview?(config: TimberConfig, buildDir: string): Promise<void>;
  waitUntil?(promise: Promise<unknown>): void;
  sendEarlyHints?(links: string[]): void;
}
```
